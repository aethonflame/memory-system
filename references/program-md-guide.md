# PROGRAM.md Guide

`PROGRAM.md` is the consolidation agent's genome.
It should contain only the durable, repeatable job description — not session-specific status.

## Good PROGRAM.md content
- What files to read
- What files to update
- Size limits
- Promotion rules
- What never to write
- Run-log expectations

## Bad PROGRAM.md content
- Today's project status
- Current blockers
- Temporary debugging notes
- Human-oriented changelog entries

## Rule of thumb
If it changes every day, it belongs in a run log or daily note.
If it should stay true for months, it belongs in PROGRAM.md.
